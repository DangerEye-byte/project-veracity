HOW IT WORKS

Three-layer verification pipeline that catches hallucinations at the semantic, numerical, and logical level.

01
Query Routing
Your query hits the Ensemble Retriever — BM25 for keyword precision combined with Qdrant vector search for semantic similarity. The best-matching ground truth document is surfaced.
02
AI Response Generation
Gemini streams a response token by token through our Node.js server. The full response is captured server-side — your Gemini API key never touches the browser.
03
3-Layer Audit
Topic relevance via semantic similarity → NLI contradiction detection via DeBERTa → verdict rendered with source reference attached.
04
Surgical Correction
Hallucinated content is struck through in red and replaced with verified text in green. Hovering reveals the exact source document.

Capabilities

BUILT TO BE UNBEATABLE

Every layer engineered to handle edge cases that break simpler approaches.

🧠
Semantic Topic Gating
Before any NLI runs, we verify the retrieved document is actually about the query. Prevents false contradictions from loosely related documents.
⚡
Ensemble Retrieval
BM25 keyword matching (0.3 weight) + Qdrant vector search (0.7 weight) combined for maximum recall across lexical and semantic queries.
🔬
DeBERTa NLI Engine
cross-encoder/nli-deberta-v3-small — a discriminative classifier. 50ms vs 5s for generative LLM approaches. Deterministic. No prompt engineering.
🎯
Ambiguity Detection
Claims too vague to verify return NEUTRAL. Partial information is never penalised as wrong. Benefit of the doubt is baked in by design.
🔐
Secure Node Proxy
Gemini API key lives in .env on the server — never exposed to the browser. All AI calls proxied server-side through Express.
🏗️
Private Knowledge Base
Your documents never leave your infrastructure. Qdrant runs locally. The entire verification pipeline is fully self-hosted and air-gap compatible.
